# Repo Family Charter

## Family

1. `Moonstone`: flagship orchestrator runtime plane
2. `Mariner`: future transport/gateway repo
3. `Surveyor`: future eval/conformance/policy repo
4. `Ranger`: future execution/runtime worker repo
5. `Pioneer`: future standards/template repo after Moonstone proves two stable slices

## Current Scope

Only `Moonstone` is active now.

## Split Criteria

1. Split to `Mariner` when transport adapters outgrow core runtime concerns.
2. Split to `Surveyor` when conformance/promotion workflows become independently versioned.
3. Split to `Ranger` when execution workers need isolated release cadence.
4. Do not create `Pioneer` before stable pattern extraction.
