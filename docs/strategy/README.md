## Tracker Rules

1. Allowed status values are `planned`, `in_progress`, `blocked`, `done`.
2. Every future action must include `evidence_ref` (use `none` only when no evidence exists yet).
3. Every `linked_milestone_id` in future actions must exist in milestones.
4. Milestone and action IDs must be unique.

## Naming Pattern

1. `YYYY-MM-DD-topic-build-plan.md`
2. `YYYY-MM-DD-topic-risk-register.md`
