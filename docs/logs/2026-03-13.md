## Module Format Decision Record (MJS vs TS)

- Decision: keep active PF-POC implementation on `.mjs`; do not introduce ad-hoc `.ts` runtime files during current product slices.
- Evidence snapshot at decision time:
  - repository extension inventory: `.mjs` = 113, `.ts` = 0
  - toolchain contract: Node ESM `.mjs` scripts in `package.json` and strict JS type-check mode (`allowJs` + `checkJs` + `noEmit`) in `tsconfig.json`
- Rationale: mixed piecemeal `.ts` adoption now would create dual-path toolchain/runtime drift and slow product-signal delivery.
- Follow-up constraint: schedule TypeScript source migration as its own milestone with explicit conversion + gate parity criteria.
