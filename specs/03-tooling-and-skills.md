# Tooling And Skills

## Local Skill Policy

Local `.codex/skills` is mandatory and has precedence over global/system skills.

Required default local skills:

- `planning-with-files`
- `writing-plans`
- `orchestrator`
- `executing-plans`
- `verification-before-completion`
- `reflexion`
- `manage-skills`

Required reinforced pack:

- `hermeneia`
- `prothesis`
- `syneidesis`
- `test-driven-development`
- `eval-suite-checklist`
- `tool-design-checklist`
- `systematic-debugging`

## Skill Governance

1. `skill-installer` is used only to bootstrap curated skills into local catalog.
2. `skill-creator` is used when repeated project workflows justify a dedicated local skill.
3. `manage-skills` audit is mandatory once per milestone and on local skill changes.
